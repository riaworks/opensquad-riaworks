# Quality Criteria: riaworks-youtube

> Critérios de qualidade para avaliação de scripts YouTube.
> Calibrados com métricas reais de 60 vídeos investigados.

---

## Critérios para YouTube Script (Long-form)

### 1. Hook (Peso: 25%)
- [ ] Hook nos primeiros 30 segundos sem saudação ("Olá pessoal")
- [ ] Cria curiosity gap que precisa de resolução
- [ ] Usa 1 dos 3 padrões comprovados: afirmação provocativa, número concreto, ou pergunta desafiadora
- [ ] Estabelece stakes claras — por que o espectador deve assistir
- **Threshold**: Mínimo 7/10. Hook fraco = rejeição automática.

### 2. Título (Peso: 20%)
- [ ] 50-70 caracteres (não truncado em search/mobile)
- [ ] Keyword primária nos primeiros 40 caracteres
- [ ] Combina curiosity gap + benefício específico
- [ ] Complementa (não repete) o thumbnail concept
- [ ] Capitalização seletiva (1-2 palavras no máximo)
- **Threshold**: Mínimo 6/10.

### 3. Estrutura e Retenção (Peso: 20%)
- [ ] 3-5 blocos temáticos com progressão lógica
- [ ] Mini-hook no início de cada bloco
- [ ] Transições com curiosidade entre blocos ("Mas tem um problema...")
- [ ] Pattern interrupts marcados a cada 45-90 segundos
- [ ] Payoff posicionado nos últimos 30% do vídeo (não entregue cedo)
- [ ] Duração entre 8-15 minutos (ou justificativa para fora dessa faixa)
- **Threshold**: Mínimo 6/10.

### 4. Conteúdo e Valor (Peso: 15%)
- [ ] Demonstração prática ou case study real (não só teoria)
- [ ] Dados específicos e verificáveis (números, percentuais, fontes)
- [ ] Pelo menos 1 insight acionável por minuto de vídeo
- [ ] Analogias para conceitos abstratos (tech/IA requer simplificação)
- **Threshold**: Mínimo 7/10.

### 5. Tom e Voz (Peso: 10%)
- [ ] Conversacional, direto, opinativo sem arrogância
- [ ] Frases curtas para fala natural (não leitura acadêmica)
- [ ] Vocabulário alinhado com público tech brasileiro
- [ ] Acentuação pt-BR correta em todo o texto
- **Threshold**: Mínimo 6/10.

### 6. CTA e Encerramento (Peso: 5%)
- [ ] CTA contextualizado (não genérico "curte e se inscreve")
- [ ] Síntese final em 3 bullets
- [ ] Teaser do próximo vídeo ou conteúdo relacionado
- **Threshold**: Mínimo 5/10.

### 7. Thumbnail Concept (Peso: 5%)
- [ ] Definido ANTES do roteiro
- [ ] 3-4 palavras de texto overlay
- [ ] Expressão facial/emoção descrita
- [ ] Alto contraste (cores vivas)
- [ ] Legível em tamanho mobile
- **Threshold**: Mínimo 6/10.

## Critérios para YouTube Shorts

### 1. Hook Instantâneo (Peso: 30%)
- [ ] Curiosity gap ou visual hook nos primeiros 2 segundos
- [ ] Text overlay bold na primeira frame (máx 8 palavras)
- [ ] Impossível de ignorar (provocação, dado chocante, visual disruptivo)
- **Threshold**: Mínimo 8/10. Hook fraco em Short = rejeição automática.

### 2. Densidade e Foco (Peso: 25%)
- [ ] 1 tópico, 1 takeaway (sem multi-point)
- [ ] 15-45 segundos (sweet spot para completion rate)
- [ ] Zero filler words (um, então, basicamente)
- [ ] Ritmo acelerado e energético
- **Threshold**: Mínimo 7/10.

### 3. Loop Design (Peso: 20%)
- [ ] Final conecta de volta ao início (visual ou narrativo)
- [ ] Incentiva replay natural
- [ ] Punchline que resolve a tensão do hook
- **Threshold**: Mínimo 6/10.

### 4. Acessibilidade (Peso: 15%)
- [ ] Text overlays ao longo de todo o Short (sound-off viewing)
- [ ] Formato nativo vertical 9:16
- [ ] "#Shorts" incluído no título ou descrição
- [ ] 3-5 hashtags relevantes
- **Threshold**: Mínimo 6/10.

### 5. Tom e Voz (Peso: 10%)
- [ ] Consistente com o canal (técnico, direto, energético)
- [ ] Acentuação pt-BR correta
- **Threshold**: Mínimo 6/10.

---

## Regras de Decisão

| Condição | Veredito |
|---|---|
| Overall ≥ 7/10, nenhum critério abaixo de 4/10 | **APPROVE** |
| Overall ≥ 7/10, critério não-crítico entre 4-6/10 | **CONDITIONAL APPROVE** |
| Overall < 7/10 | **REJECT** |
| Hook abaixo de 6/10 | **REJECT** (hard trigger) |
| Título abaixo de 5/10 | **REJECT** (hard trigger) |
| 3+ ciclos de revisão com mesmos problemas | **ESCALATE** ao usuário |
